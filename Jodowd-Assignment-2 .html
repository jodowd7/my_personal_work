I just couldn't narrow down a selection to one technique. I enjoy exploring and seeing the possible ways of outputting visual results. I also thought only one visualization wasn't
the best way to sample a library unfamiliar to me such as Seaborn. So I chose four different techniques: scatterplot, violin plot, the simple bar plot, and the heat map.

My first chosen technique was the scatterplot. I chose it for three reasons. The first reason was that it's new to me. I have never used it before. The second reason was it looked
interesting. The third reason is because it fit the framework of how to properly express the data within my chosen data set. The data set I found was quantitative in nature. It depicted
the number of soldier casualties overall and per country in Afghanistan. Scatterplot is great at plotting this data by using individual points or dots to represent the number of deaths
per country per year. It does a remarkable job of pointing out patterns. It visually helped me to see quickly which country lost the most soldiers in the war against terrorism in Afghanistan.
It is often difficult to see the big picture when merely examining the data set without a good visualization technique. Through the scatterplot, one sees immediately a visual representation
that tells the true story behind the data.

My second chosen technique was the violin plot. It works to graphically represent all at once various components of each data record such as the highest point,the lowest point, the mean, and
probability density.

My third chosen technique was the bar plot which allows one to quickly see differences among the categories in an extremely clear and visually aesthetic appearance. Bar plots represent categorical
values in a rectangular fashion from beginning values to the final accumulated value. This is perfect for my sensitive data set which takes into account visually the death of every single one
of our soldiers. Instead of just a final count, it adds up each life that was lost. Visually this is symbolic of the overall cost of each nation. Each soldier's life matters and is counted as
an individual and rolled into a final count at the end of the bar.

My fourth chosen technique was the heat map. The heat map is used to depict certain truths in a visual way using colors; for example, if using viridis as the CMAP setting, a progression from
darker to lighter colors can represent a certain degree of increasing intensity or increasing values, or if using binary, a progression of lighter to darker shades represent increasing values.
Towards the end of this notebook, one field value was used in a heat map over time. Using CMAP = viridis, the lighter colors represent the most caualties lost by the U.S. soldiers while the
darker colors represent the least casualties. Using CMAP = binary, the darker shades represent the most casualties.

time. Another technique closely aligned to the scatterplot is the line plot as it does basically the same thing plotting points on the graph; however, it also connects

NOTE: Seaborn has a parameter called SIZE. One thing I noticed though is that if you use SIZE on MULTIPLE y values, it will distort the colors in the legend.
The legend was created using MATPLOTLIB. The SIZE of the dot was created using SEABORN and thus, this appears to be an incompatibility between MATPLOTLIB and Seaborn.
Therefore, I decided to use SIZE only on the last Y value of the country of France. I chose to lessen the size of each scatter plot for France
because they were harder to see due to other country dots with similar values as France.
I am quite impressed with SEABORN over all as it provides a wide variety of parameters to manipulate your output visuals to make a point about something.
